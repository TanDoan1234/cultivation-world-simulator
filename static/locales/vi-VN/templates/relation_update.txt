Yêu cầu phân tích:
1. Dựa trên hồ sơ tương tác, phân tích xem mức độ giao tiếp giữa hai người đang diễn ra như thế nào.
2. Nó có đáp ứng đủ các điều kiện để tạo một trạng thái quan hệ mới hoặc hủy bỏ quan hệ cũ như trong bảng quy tắc không?
3. Phân tích xem có nên kích hoạt biến động quan hệ hay không; sự thêm vào hay phá vỡ quan hệ phải thoả mãn các điều kiện quy định.

Trả về duy nhất bằng định dạng JSON:
{{
    "analysis": "...", // Một phân tích ngắn gọn bằng tiếng Việt về lý lẽ, nêu rõ vì sao thay đổi hoặc không thay đổi quan hệ.
    "changed": true | false, // Xác nhận có hay không sự thay đổi mối quan hệ.
    "change_type": "ADD" | "REMOVE", // Loại biến động thêm mới hoặc thủ tiêu. Có thể bỏ qua nếu changed là false.
    "relation": "LOVERS" | "FRIEND" | "ENEMY" | "MASTER" ... (Bắt buộc phải giữ NGUYÊN tên IN HOA tiếng Anh), // Quan hệ liên quan. Bỏ qua nếu changed=false. Lưu ý đây là trạng thái của {avatar_a_name} so với {avatar_b_name}. Ví dụ, nếu kết quả trả về MASTER, nghĩa là A trở thành sư phụ của B.
    "reason": "..." // Mô tả ngắn gọn nguyên nhân bằng tiếng Việt, dưới dạng một từ hay cụm từ ngắn gọn trong mười chữ, như "Tri âm tri kỷ". Bỏ qua nếu changed=false.
}}
